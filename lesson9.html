Semantic Elements

  • HTML achieves document organization through semantic markup
  • Semantic elements defines a meaning rather than a generic division– also makes it accessible to screen readers and other assistive technologies
  • There should be a hierarchy of headers (h1 to h6 are semantic tags)
  • There should be a hierarchy of headers (h1 to h6 are semantic tags)
  • Only one h1 tag should be used within in each page
  • You can have as many h2 tags as you need, there should be at least one h2 for every h3; it is all like an outline
  • You don't want more than three levels of headings within a page
  • There can be one header element for every page but it has to follow the heading hierarchy– it is located at the top of the web page
  • <nav> is within the header
  • <nav> elements are all semantic elements– within the nav provides links to other pages
  • There can be more than one <nav> elements within a page
  • <nav> be labeled using an aria-label attribute, when there is no heading for the <nav> element
  • <hr> (horizontal rules) can have a semantic meaning but it is not a semantic tag- represents a topic change
  • There is only one <main> element per page
  • There can be one footer element per page just like the header element
  • <li> are also semantic elements
  • A <div> element is not semantic because it is also a section it does explain what the section is talking about
  • <section> & <article> are semantic tags
  • <article> have a topic and might have content separate from the rest of the page
  • <details> and <summary> work together and are semantic elements
  • <summary> is a subsection to details
  • <details> let you hide content and open it with the press of an arrow

The Div Tag

HTML4 introduced the <div> tag, which allowed web developers to create divisions, or sections to their pages. Web pages soon had many divs containing different types of content. While useful, overuse of div elements can make it difficult to understand how the page is structured.
